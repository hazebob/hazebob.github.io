Next.js 13+에서 도입된 App Router는 기존의 Pages Router와 비교하여 많은 장점을 제공합니다.
서버 컴포넌트를 기본으로 사용하며, 더 나은 성능과 개발 경험을 제공합니다.

## 주요 기능

### 1. 서버 컴포넌트 기본 지원

```tsx
// app/page.tsx
export default async function Page() {
  const data = await fetchData() // 서버에서 직접 데이터 페칭
  return <main>{/* 데이터 렌더링 */}</main>
}
```

### 2. 향상된 데이터 페칭

- 컴포넌트 레벨 데이터 페칭
- 자동 요청 중복 제거
- 스트리밍 지원

### 3. 레이아웃과 중첩 레이아웃

```tsx
// app/blog/layout.tsx
export default function BlogLayout({ children }) {
  return (
    <div>
      <nav>블로그 네비게이션</nav>
      {children}
    </div>
  )
}
```

### 4. 스트리밍과 Suspense 지원

```tsx
import { Suspense } from 'react'

export default function Page() {
  return (
    <Suspense fallback={<Loading />}>
      <SlowComponent />
    </Suspense>
  )
}
```

## 마이그레이션 고려사항

기존 Pages Router에서 App Router로 마이그레이션할 때는 다음 사항들을 고려해야 합니다:

1. 서버/클라이언트 컴포넌트 구분
2. 데이터 페칭 방식 변경
3. 라우팅 구조 재설계
4. 메타데이터 처리 방식 변경
