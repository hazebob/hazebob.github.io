## 작업 순서 
1. 오늘 할 일 정의 
  1. 하루 단위의 일은 노트에 수기로 작성
  2. 장기적인 프로젝트는 /docs/에 마크다운으로 작성
2. 새로운 프로젝트, 태스크 단위로 pi agent 세션 시작
3. 하려고 하는 일을 말해주면서 /docs/에 정리하도록 함
4. agent 작업 시작 지시 
5. git diff를 체크하며 변경 사항 인지
6. '구현 - diff 확인 - 개선, 재지시'의 사이클 반복 
7. diff를 기준으로 1차 코드 리뷰 
8. 커밋 - skill로 적당한 커밋 메세지 쓰도록
9. 필요하면 soft reset 하여 의미 단위로 묶어서 진행 
10. 각 지시 사이클, 커밋 단위에 hook이 걸려 있어 작업 내용을 요약 기록하도록 함. ex) 2026-06-06-ABC_project_list_and_filter.md 
  1. 이건 수시로 이뤄지고 있어서 기록물을 따로 읽지 않음. 즉, 기록은 내 인지 밖에서 계속 진행  
11. 작업 기록물은 하루에 한 번 그날의 것을 정리하여 history 문서로 작성, 그루핑
12. history 기반으로 다음날 할 작업, 미해결 과제, 아이디어를 작성하여 아침에 브리핑 받음 
  1. 이건 프로젝트 바깥 영역에서 Hermes agent가 진행
13. 가끔 수기로 코딩하거나 개발 아이디어들도 위의 요약 기록에 하도록 함. 끄적임 --> '이 부분 기록합니다'

## 규칙 
- 모든 프롬프트는 존댓말로 작성
- 단, 스크랩, 저장, 커밋, 기록과 같은 내용은 짧은 명령어로 진행함. 

## 얻은 것
- 10년 넘게 못하던 '업무일지'의 산을 드디어 넘게 된 것 같다.

- 기록 그 자체로서의 의미도 있지만 나의 인지 복원, 에이전트의 세션 인지 복원의 효과가 크다.
